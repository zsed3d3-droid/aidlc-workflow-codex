# DESIGN.md Evaluation Rubric

Score by visible design fidelity, not only token presence.

## Blocking

- Uses a different design movement than `DESIGN.md`.
- Primary action color, typography, or shape contradicts core tokens.
- Text/background contrast fails for normal text.
- Mobile layout overlaps, clips, or loses primary actions.
- A key component state is absent for an interactive control.

## Major

- Spacing rhythm visibly differs from the scale.
- Components use correct colors but wrong shape, density, or hierarchy.
- Accent color appears in decorative contexts forbidden by `DESIGN.md`.
- Typography sizes/weights drift from the hierarchy.
- Cards, panels, or shadows use a depth model not described by `DESIGN.md`.

## Minor

- Slight alignment inconsistency.
- Missing polish on secondary states.
- Copy length causes awkward but recoverable wrapping.
- Non-critical component variant should be added to `DESIGN.md`.

## Review Axes

- Design movement fit.
- Token and component fidelity.
- Color role discipline.
- Typography hierarchy.
- Layout and spacing.
- Elevation/depth model.
- Shape language.
- Responsive behavior.
- Accessibility.
- Domain-specific UI clarity.
