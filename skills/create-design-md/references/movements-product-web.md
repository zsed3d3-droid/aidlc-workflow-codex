# Product/Web Design Movements

Use for general web apps, SaaS, dashboards, portfolios, ecommerce, technical tools, and brand/product sites. Keep options practical and implementation-aware.

## Movement Entries

### Editorial Minimalism

- Definition: Content-first UI using typography, whitespace, and quiet hierarchy instead of decoration.
- Visual cues: Restrained palette, strong type scale, large margins, sparse surfaces, few decorative cards.
- Best fit: Publishing, docs, portfolios, premium content, trust-heavy pages.
- Use carefully / avoid: Poor for dense controls, games, or many simultaneous actions.
- Request UI rationale hints: Choose when reading, credibility, and calm focus beat interaction density; tradeoff is lower expressiveness.

### Productivist Dashboard

- Definition: Dense but calm operational UI for repeated comparison, monitoring, and task execution.
- Visual cues: Strong grids, compact panels, muted surfaces, explicit status colors, predictable navigation.
- Best fit: SaaS tools, admin consoles, CRM, analytics, infrastructure dashboards.
- Use carefully / avoid: Avoid oversized hero/card marketing composition.
- Request UI rationale hints: Choose when users scan, compare, and recover from errors often; tradeoff is less emotional brand personality.

### Constructivist Learning

- Definition: Geometric, high-contrast learning UI that makes concepts and progress concrete.
- Visual cues: Modular teaching panels, bold hierarchy, limited primary colors, visible progress states.
- Best fit: Education, tutorials, onboarding, drills.
- Use carefully / avoid: Avoid for quiet enterprise polish or subtle luxury.
- Request UI rationale hints: Choose when feedback and step-by-step focus matter; tradeoff is a more instructional tone.

### Bauhaus Utility

- Definition: Function-led UI reduced to primary shapes, limited colors, and purposeful spacing.
- Visual cues: Circles, squares, lines, primary accents, primitive controls, minimal ornament.
- Best fit: Tools, hardware-adjacent apps, explainers, educational utilities.
- Use carefully / avoid: Avoid decorative shape overload.
- Request UI rationale hints: Choose when the app needs directness and teachable affordances; tradeoff is reduced warmth.

### Swiss / International

- Definition: Grid-first UI built on neutral typography, strict alignment, and disciplined hierarchy.
- Visual cues: Modular grids, left alignment, neutral sans type, minimal color, precise spacing.
- Best fit: Docs, institutional products, developer tools, finance content.
- Use carefully / avoid: Avoid when strong playfulness or expressive consumer branding is needed.
- Request UI rationale hints: Choose when clarity and credibility matter; tradeoff is less emotional distinctiveness.

### Neobrutalist Educational

- Definition: Playful high-contrast UI with visible borders, chunky controls, and deliberately simple shapes.
- Visual cues: Thick strokes, hard panels, bold labels, bright contrasts, obvious states.
- Best fit: Youth learning, creative tools, playful onboarding.
- Use carefully / avoid: Avoid serious finance, medical, or executive workflows.
- Request UI rationale hints: Choose when affordances must be unmistakable; tradeoff is lower refinement.

### Soft Tactile Mobile

- Definition: Touch-first UI emphasizing warmth, rounded surfaces, gentle depth, and comfortable controls.
- Visual cues: Large tap targets, soft radii, warm neutrals, gentle shadows, forgiving spacing.
- Best fit: Mobile apps, wellness, personal finance, habit tracking, family apps.
- Use carefully / avoid: Avoid for dense technical tooling.
- Request UI rationale hints: Choose when safe, friendly tapping matters; tradeoff is lower data density.

### Game-like Companion UI

- Definition: Companion-style product UI with expressive assets, reward feedback, and restrained motion.
- Visual cues: Characters, progress rewards, stateful feedback, bright controlled accents.
- Best fit: Coaching, habit loops, gamified onboarding, companion apps.
- Use carefully / avoid: Avoid decorative reward clutter and unclear primary tasks.
- Request UI rationale hints: Choose when motivation and attachment matter; tradeoff is asset/motion cost.

### Dark Technical Cockpit

- Definition: Dark, data-dense tooling UI for technical monitoring and operational control.
- Visual cues: Dark surface ladder, sparse accent, monospaced labels, charts, logs, compact controls.
- Best fit: Devtools, observability, security operations, AI infrastructure.
- Use carefully / avoid: Avoid consumer apps and long reading unless fatigue is managed.
- Request UI rationale hints: Choose when dense technical state needs focus; tradeoff is accessibility/fatigue risk.

### Pastel Editorial Blocks

- Definition: Poster-like editorial UI with a white base, large pastel blocks, and clear content hierarchy.
- Visual cues: Large panels, soft color fields, display type, section rhythm, minimal chrome.
- Best fit: Marketing pages, portfolios, events, friendly consumer landing pages.
- Use carefully / avoid: Avoid heavy operational tooling.
- Request UI rationale hints: Choose when approachable hierarchy matters without complex assets; tradeoff is lower task density.

### Skeuomorphism

- Definition: Realism-driven UI mapping physical materials, texture, light, and depth into digital controls.
- Visual cues: Realistic buttons, metal/glass/leather cues, highlights, physical shadows, tactile knobs.
- Best fit: Audio plugins, smart-home controls, creative production tools, hardware metaphors.
- Use carefully / avoid: Avoid general web apps where realism adds weight or looks dated.
- Request UI rationale hints: Choose when users understand a physical metaphor; tradeoff is asset complexity.

### Glassmorphism

- Definition: Frosted-glass UI creating depth through translucent blurred layers and light borders.
- Visual cues: `backdrop-filter: blur`, translucent panels, subtle borders, layered surfaces.
- Best fit: OS-like overlays, media controls, premium ambient dashboards.
- Use carefully / avoid: Contrast, readability, and performance risk.
- Request UI rationale hints: Choose when layered context matters; tradeoff is accessibility/performance.

### Neumorphism / Soft UI

- Definition: Soft extruded-surface UI combining flat simplicity with light/shadow relief.
- Visual cues: Same-color surfaces, paired light/dark shadows, pressed/raised states.
- Best fit: Small tactile controls, experimental portfolios, non-critical secondary surfaces.
- Use carefully / avoid: Avoid primary nav, forms, CTAs, and accessibility-sensitive flows.
- Request UI rationale hints: Choose rarely for small tactile controls; tradeoff is poor affordance clarity.

### Claymorphism

- Definition: Friendly soft-3D UI with higher contrast, rounded clay-like volume, and playful mass.
- Visual cues: Puffy shapes, large rounded components, inner/outer shadows, bright pastels.
- Best fit: Youth products, events, friendly fintech moments, metaverse/game-adjacent UI.
- Use carefully / avoid: Avoid dense enterprise and serious text-heavy products.
- Request UI rationale hints: Choose when cute tactile warmth matters; tradeoff is lower seriousness.

### Material Design

- Definition: Google's paper-inspired design system with formal elevation, motion, and component rules.
- Visual cues: Grid layouts, quantified shadows, Z-axis elevation, FABs, ripple/motion feedback.
- Best fit: Android apps, cross-platform products, admin panels using Material libraries.
- Use carefully / avoid: Can look generic when brand needs distinctiveness.
- Request UI rationale hints: Choose for implementation speed and mature components; tradeoff is less unique identity.

### Flat 2.0 / Semi-flat

- Definition: Pragmatic flat UI that restores affordance through subtle depth and gradients.
- Visual cues: Flat shapes, minimal icons, subtle shadows, slight CTA/card gradients, clear focus states.
- Best fit: Mainstream SaaS, CRUD apps, collaboration tools, productivity products.
- Use carefully / avoid: Avoid generic output by defining distinctive tokens.
- Request UI rationale hints: Choose for familiar usability and broad adoption; tradeoff is weaker memorability.

### Bento Grid

- Definition: Modular layout dividing content into different-sized rounded grid cards.
- Visual cues: Variable card sizes, rounded rectangles, fine borders, independent card backgrounds.
- Best fit: Product overviews, dashboards, portfolio summaries, modular feature displays.
- Use carefully / avoid: Avoid nested-card clutter and poor mobile collapse.
- Request UI rationale hints: Choose when many modules must be visible; tradeoff is layout complexity.

### Scannable Bento-Minimalism

- Definition: Restrained Bento variant optimized for fast scanning rather than decorative modularity.
- Visual cues: Limited palette, strict card hierarchy, embedded data visuals, minimal copy.
- Best fit: Hardware showcases, product archives, executive summaries.
- Use carefully / avoid: Avoid when narrative reading matters.
- Request UI rationale hints: Choose when users must grasp many modules quickly; tradeoff is less storytelling.

### Retro Cyberpunk / Vaporwave

- Definition: Subculture UI from arcade screens, CRT interfaces, neon sci-fi, glitches, and dystopian futures.
- Visual cues: Neon pink/violet/green, scanlines, pixel elements, glitch animation, dark grids.
- Best fit: Indie games, streamer overlays, Web3 launches, music events.
- Use carefully / avoid: Avoid mainstream productivity, healthcare, finance.
- Request UI rationale hints: Choose when atmosphere outranks calm utility; tradeoff is visual noise.

### Acid Graphic / Y2K Techno

- Definition: Millennium-futurist UI mixing Y2K chrome, metallic textures, distorted 3D objects, and high-energy graphics.
- Visual cues: Chrome blobs, metallic surfaces, rainbow gradients, distorted type, layout warping.
- Best fit: Fashion ecommerce, music promos, streetwear, youth culture launches.
- Use carefully / avoid: Avoid serious operational workflows and long-term neutrality needs.
- Request UI rationale hints: Choose for trend signal and attention capture; tradeoff is fast visual aging.

### Corporate Memphis / Alegria

- Definition: Flat illustration UI using geometric postmodern forms, simplified people, and friendly abstract scenes.
- Visual cues: Long-limbed characters, pastel/primary color blocks, distorted perspective, empty-state illustrations.
- Best fit: Onboarding, empty states, service explainers, friendly consumer education.
- Use carefully / avoid: Overused; customize heavily to avoid generic startup feel.
- Request UI rationale hints: Choose when abstract concepts need friendly explanation; tradeoff is genericness.

### Maximalist Rule-breaking / Anti-Design

- Definition: Deliberately disruptive UI that breaks grids, overlaps content, and challenges readability.
- Visual cues: Huge overlapping type, obscured images, asymmetry, horizontal scroll, rough alignment.
- Best fit: Creative agencies, art museums, fashion houses, experimental campaigns.
- Use carefully / avoid: High usability risk for transactional, operational, or accessibility-critical products.
- Request UI rationale hints: Choose when attention shock and cultural positioning matter; tradeoff is comprehension risk.

### Organic Eco-Minimalism / Nature Distilled

- Definition: Natural UI translating sustainability, wellness, and earthiness into calm digital systems.
- Visual cues: Earth tones, muted greens, sand/off-white, paper grain, organic curves, gentle type pairings.
- Best fit: Eco brands, vegan cosmetics, wellness, meditation, ethical fashion.
- Use carefully / avoid: Avoid one-note beige/earth-tone dominance and low contrast.
- Request UI rationale hints: Choose when calm natural trust matters; tradeoff is lower technical sharpness.

### Human-Crafted Imperfection

- Definition: Handmade-modern UI that counters template polish with deliberate human irregularity.
- Visual cues: Uneven borders, hand-drawn SVGs, paper noise, irregular component sizes.
- Best fit: Creative studios, independent makers, art spaces, craft commerce.
- Use carefully / avoid: Avoid strict enterprise workflows and heavily standardized component systems.
- Request UI rationale hints: Choose when originality and human authorship matter; tradeoff is harder systematization.

### Kinetic Typography Layout

- Definition: Type-led motion UI where large typography becomes the primary visual structure.
- Visual cues: Oversized text, variable fonts, scroll-reactive type, marquee motion.
- Best fit: Launch pages, music/streaming promos, type foundries, brand campaigns.
- Use carefully / avoid: Motion/readability risk for long-form content and forms.
- Request UI rationale hints: Choose when the message is the hero asset; tradeoff is motion complexity.

### Chromatic Maximalism

- Definition: Color-forward UI that rejects restraint and fills the interface with intense expressive palettes.
- Visual cues: Clashing complements, fluid gradients, high-chroma surfaces, explosive hover states.
- Best fit: Entertainment, creator tools, music/video products, agency sites.
- Use carefully / avoid: Visual-noise and fatigue risk for data-heavy tools.
- Request UI rationale hints: Choose when emotional energy matters most; tradeoff is harder accessibility tuning.

### Aurora / Fluid Glow

- Definition: Dark ambient UI using luminous organic gradients and glow to signal premium technical sophistication.
- Visual cues: Deep dark surfaces, blurred gradient glow, floating light fields, subtle pointer-reactive motion.
- Best fit: AI SaaS, developer tools, premium technical landing pages.
- Use carefully / avoid: Avoid decorative orbs; keep glow tied to hierarchy.
- Request UI rationale hints: Choose when advanced atmosphere matters without cockpit density; tradeoff is AI-tool sameness.

### Floating Canvas / Immersive Canvas

- Definition: Spatial web UI that breaks vertical page boundaries and lets users explore a large interactive canvas.
- Visual cues: Pan/drag navigation, horizontal/infinite scroll, parallax, 2D/3D mix, WebGL/Spline-like scenes.
- Best fit: Brand showcases, interactive portfolios, spatial diagrams, award-style websites.
- Use carefully / avoid: High navigation/accessibility risk for CRUD, forms, dashboards.
- Request UI rationale hints: Choose when exploration is the product experience; tradeoff is engineering complexity.

### Zero-UI / Contextual Ambient

- Definition: Minimal interaction UI hiding chrome until context, gesture, or command intent requires it.
- Visual cues: Sparse nav, command palettes, progressive disclosure, contextual panels, keyboard-first flows.
- Best fit: Power-user productivity, AI assistants, developer tools, focused command workflows.
- Use carefully / avoid: Avoid novice/touch-first products needing persistent affordances.
- Request UI rationale hints: Choose when experts value speed and low chrome; tradeoff is discoverability.
