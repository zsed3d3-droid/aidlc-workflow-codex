# Product/Web 디자인 사조

General web app, SaaS, dashboard, portfolio, ecommerce, technical tool, brand/product site에 사용한다. option은 practical하고 implementation-aware하게 유지한다.

## Movement Entries

### Editorial Minimalism

- Definition: decoration 대신 typography, whitespace, quiet hierarchy를 사용하는 content-first UI.
- Visual cues: restrained palette, strong type scale, large margin, sparse surface, decorative card 최소화.
- Best fit: Publishing, docs, portfolio, premium content, trust-heavy page.
- Use carefully / avoid: dense control, game, simultaneous action이 많은 UI에는 부적합하다.
- Request UI rationale hints: reading, credibility, calm focus가 interaction density보다 중요할 때 선택한다. tradeoff는 낮은 expressiveness.

### Productivist Dashboard

- Definition: 반복적인 comparison, monitoring, task execution을 위한 dense하지만 calm한 operational UI.
- Visual cues: strong grid, compact panel, muted surface, explicit status color, predictable navigation.
- Best fit: SaaS tool, admin console, CRM, analytics, infrastructure dashboard.
- Use carefully / avoid: oversized hero/card marketing composition을 피한다.
- Request UI rationale hints: user가 자주 scan, compare, error recovery를 해야 할 때 선택한다. tradeoff는 낮은 emotional brand personality.

### Constructivist Learning

- Definition: concept와 progress를 concrete하게 만드는 geometric high-contrast learning UI.
- Visual cues: modular teaching panel, bold hierarchy, 제한된 primary color, visible progress state.
- Best fit: Education, tutorial, onboarding, drill.
- Use carefully / avoid: quiet enterprise polish 또는 subtle luxury에는 피한다.
- Request UI rationale hints: feedback과 step-by-step focus가 중요할 때 선택한다. tradeoff는 더 instructional한 tone.

### Bauhaus Utility

- Definition: primary shape, limited color, purposeful spacing으로 줄인 function-led UI.
- Visual cues: circle, square, line, primary accent, primitive control, minimal ornament.
- Best fit: Tool, hardware-adjacent app, explainer, educational utility.
- Use carefully / avoid: decorative shape overload를 피한다.
- Request UI rationale hints: app에 directness와 teachable affordance가 필요할 때 선택한다. tradeoff는 warmth 감소.

### Swiss / International

- Definition: neutral typography, strict alignment, disciplined hierarchy를 기반으로 한 grid-first UI.
- Visual cues: modular grid, left alignment, neutral sans type, minimal color, precise spacing.
- Best fit: Docs, institutional product, developer tool, finance content.
- Use carefully / avoid: 강한 playfulness 또는 expressive consumer branding이 필요하면 피한다.
- Request UI rationale hints: clarity와 credibility가 중요할 때 선택한다. tradeoff는 낮은 emotional distinctiveness.

### Neobrutalist Educational

- Definition: visible border, chunky control, deliberately simple shape를 쓰는 playful high-contrast UI.
- Visual cues: thick stroke, hard panel, bold label, bright contrast, obvious state.
- Best fit: Youth learning, creative tool, playful onboarding.
- Use carefully / avoid: serious finance, medical, executive workflow에는 피한다.
- Request UI rationale hints: affordance가 아주 명확해야 할 때 선택한다. tradeoff는 낮은 refinement.

### Soft Tactile Mobile

- Definition: warmth, rounded surface, gentle depth, comfortable control을 강조하는 touch-first UI.
- Visual cues: large tap target, soft radius, warm neutral, gentle shadow, forgiving spacing.
- Best fit: Mobile app, wellness, personal finance, habit tracking, family app.
- Use carefully / avoid: dense technical tooling에는 피한다.
- Request UI rationale hints: safe하고 friendly한 tapping이 중요할 때 선택한다. tradeoff는 낮은 data density.

### Game-like Companion UI

- Definition: expressive asset, reward feedback, restrained motion을 쓰는 companion-style product UI.
- Visual cues: character, progress reward, stateful feedback, bright controlled accent.
- Best fit: Coaching, habit loop, gamified onboarding, companion app.
- Use carefully / avoid: decorative reward clutter와 unclear primary task를 피한다.
- Request UI rationale hints: motivation과 attachment가 중요할 때 선택한다. tradeoff는 asset/motion cost.

### Dark Technical Cockpit

- Definition: technical monitoring과 operational control을 위한 dark data-dense tooling UI.
- Visual cues: dark surface ladder, sparse accent, monospaced label, chart, log, compact control.
- Best fit: Devtool, observability, security operation, AI infrastructure.
- Use carefully / avoid: consumer app과 long reading에는 fatigue 관리가 없으면 피한다.
- Request UI rationale hints: dense technical state에 focus가 필요할 때 선택한다. tradeoff는 accessibility/fatigue risk.

### Pastel Editorial Blocks

- Definition: white base, large pastel block, clear content hierarchy를 쓰는 poster-like editorial UI.
- Visual cues: large panel, soft color field, display type, section rhythm, minimal chrome.
- Best fit: Marketing page, portfolio, event, friendly consumer landing page.
- Use carefully / avoid: heavy operational tooling에는 피한다.
- Request UI rationale hints: complex asset 없이 approachable hierarchy가 중요할 때 선택한다. tradeoff는 낮은 task density.

### Skeuomorphism

- Definition: physical material, texture, light, depth를 digital control에 mapping하는 realism-driven UI.
- Visual cues: realistic button, metal/glass/leather cue, highlight, physical shadow, tactile knob.
- Best fit: Audio plugin, smart-home control, creative production tool, hardware metaphor.
- Use carefully / avoid: realism이 weight를 더하거나 dated하게 보이는 general web app에는 피한다.
- Request UI rationale hints: user가 physical metaphor를 이해할 때 선택한다. tradeoff는 asset complexity.

### Glassmorphism

- Definition: translucent blurred layer와 light border로 depth를 만드는 frosted-glass UI.
- Visual cues: `backdrop-filter: blur`, translucent panel, subtle border, layered surface.
- Best fit: OS-like overlay, media control, premium ambient dashboard.
- Use carefully / avoid: contrast, readability, performance risk가 있다.
- Request UI rationale hints: layered context가 중요할 때 선택한다. tradeoff는 accessibility/performance.

### Neumorphism / Soft UI

- Definition: flat simplicity와 light/shadow relief를 결합한 soft extruded-surface UI.
- Visual cues: same-color surface, paired light/dark shadow, pressed/raised state.
- Best fit: small tactile control, experimental portfolio, non-critical secondary surface.
- Use carefully / avoid: primary nav, form, CTA, accessibility-sensitive flow에는 피한다.
- Request UI rationale hints: small tactile control에 드물게 선택한다. tradeoff는 poor affordance clarity.

### Claymorphism

- Definition: 높은 contrast, rounded clay-like volume, playful mass를 쓰는 friendly soft-3D UI.
- Visual cues: puffy shape, large rounded component, inner/outer shadow, bright pastel.
- Best fit: Youth product, event, friendly fintech moment, metaverse/game-adjacent UI.
- Use carefully / avoid: dense enterprise와 serious text-heavy product에는 피한다.
- Request UI rationale hints: cute tactile warmth가 중요할 때 선택한다. tradeoff는 낮은 seriousness.

### Material Design

- Definition: formal elevation, motion, component rule을 가진 Google의 paper-inspired design system.
- Visual cues: grid layout, quantified shadow, Z-axis elevation, FAB, ripple/motion feedback.
- Best fit: Android app, cross-platform product, Material library를 쓰는 admin panel.
- Use carefully / avoid: brand distinctiveness가 필요하면 generic하게 보일 수 있다.
- Request UI rationale hints: implementation speed와 mature component가 중요할 때 선택한다. tradeoff는 낮은 unique identity.

### Flat 2.0 / Semi-flat

- Definition: subtle depth와 gradient로 affordance를 회복한 pragmatic flat UI.
- Visual cues: flat shape, minimal icon, subtle shadow, slight CTA/card gradient, clear focus state.
- Best fit: Mainstream SaaS, CRUD app, collaboration tool, productivity product.
- Use carefully / avoid: distinctive token을 정의해 generic output을 피한다.
- Request UI rationale hints: familiar usability와 broad adoption이 중요할 때 선택한다. tradeoff는 약한 memorability.

### Bento Grid

- Definition: content를 different-sized rounded grid card로 나누는 modular layout.
- Visual cues: variable card size, rounded rectangle, fine border, independent card background.
- Best fit: Product overview, dashboard, portfolio summary, modular feature display.
- Use carefully / avoid: nested-card clutter와 poor mobile collapse를 피한다.
- Request UI rationale hints: 많은 module을 한 화면에 보여야 할 때 선택한다. tradeoff는 layout complexity.

### Scannable Bento-Minimalism

- Definition: decorative modularity보다 fast scanning에 최적화된 restrained Bento variant.

### Acid Graphic / Y2K Techno

- Visual cues: chrome blob, metallic surface, rainbow gradient, distorted type, layout warping.

### Corporate Memphis / Alegria

- Definition: geometric postmodern form, simplified people, friendly abstract scene을 쓰는 flat illustration UI.
- Visual cues: long-limbed character, pastel/primary color block, distorted perspective, empty-state illustration.

### Maximalist Rule-breaking / Anti-Design

- Definition: grid를 깨고 content를 overlap하며 readability에 도전하는 deliberately disruptive UI.

### Floating Canvas / Immersive Canvas

- Definition: vertical page boundary를 깨고 large interactive canvas를 탐색하게 하는 spatial web UI.

- Use carefully / avoid: CRUD, form, dashboard에는 high navigation/accessibility risk가 있다.
- Request UI rationale hints: exploration 자체가 product experience일 때 선택한다. tradeoff는 engineering complexity.

### Zero-UI / Contextual Ambient

- Definition: context, gesture, command intent가 필요할 때까지 chrome을 숨기는 minimal interaction UI.
- Visual cues: sparse nav, command palette, progressive disclosure, contextual panel, keyboard-first flow.
- Best fit: Power-user productivity, AI assistant, developer tool, focused command workflow.
- Use carefully / avoid: persistent affordance가 필요한 novice/touch-first product에는 피한다.
- Request UI rationale hints: expert가 speed와 low chrome을 중시할 때 선택한다. tradeoff는 discoverability.
